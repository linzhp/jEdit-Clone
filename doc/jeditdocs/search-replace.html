jEdit has a very powerful search and replace system. It supports multi-file searching and regular expressions, and also a grep-like "HyperSearch" feature. This chapter covers it in detail.

Searching for Text

The "Find" field in the tool bar can be used to search for text. A search string can be entered by clicking the the field, or by invoking Search>Find (keyboard equivalent: Control-F). Pressing Enter in the "Find" field will locate the next instance of the currently entered string. If Enter is pressed while no string is entered, the Search and Replace dialog box will be displayed. Pressing Escape in the "Find" field will move the focus back to the main text area. The find field stores previously entered strings; see the section called History Text Fields.

The three buttons to the right of the find field can be used to toggle case insensitive searching, regular expressions, and multiple file searching, respectively. Pressing the first two will toggle the respective value, and pressing the multiple file search button will display the multifile search options dialog box; see the section called Searching in Multiple Files for details.

Figure 7-1. The QuickSearch text field and controls

Search>Find Next (keyboard equivalent: Control-G) locates the next instance of the search string.

Search>Find Selection (keyboard equivalent: Control-H) moves the keyboard focus to the "Find" field in the tool bar and inserts any text currently selected in the main text area. Enter can then be pressed to search for that text.

Note: If the tool bar is hidden, the Find and Find Selection commands will display the Search and Replace dialog box instead.
