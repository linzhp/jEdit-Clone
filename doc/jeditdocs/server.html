Chapter 5. Advanced Text Editing

The Edit Server

When jEdit is started, it attempts to connect to another running jEdit instance. If the connection was successful, it transfers control to that running instance, passing it any necessary file names. Only if no other running instance can be found does jEdit start up for real. This saves startup time if another instance is already running.

The edit server is implemented using TCP/IP sockets. When a server is started, it picks a random port number and authorization key and writes it to a port file (usually in a well-known location). Client instances of jEdit then read the port file and connect to the specified port, passing it the authorization key.

The authorization key is required to prevent malicious remote attackers from opening files on your compiler.

The -noserver command line switch disables the server features entirely; no connection will be attempted, and no sever will be started. The -server command line switch can be used to change the file where port information is stored. This can be used when you want to run more than one copy of jEdit at once. For example: 

$ jedit -server=my-server-2

The same -server parameter must be passed to both the server and client.

All jEdit command line parameters except those that affect initial startup (-nosettings, -settings and -nosplash) are handled appropriately by the server.

In some circumstances, it might be desirable for the client instance to open the files in an existing view, rather than creating a new one. The -reuseview switch can be used for that purpose.
