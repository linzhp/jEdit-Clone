jEdit goes far beyond the basics when it comes to text editing; this chapter covers some more advanced features.

Working With Words

jEdit has a variety of commands for working with words. In jEdit, a word is a run of text delimited by non-alphanumeric characters.

A word can be selected by double-clicking with the mouse. Another way to select a word is with the Edit>Selection>Select Word command (keyboard equivalent: Control-E W).

Control-Left moves the caret to the previous word. Control-Right moves the caret to the next word. Holding down shift in addition to one of these keys will extend the selection, instead of moving the caret.

Pressing Control in addition to Backspace or Delete deletes the word before and in front of the caret, respectively.

Edit>Word Count displays a dialog box with the number of characters, words and lines in the current buffer.

Figure 5-1. The word count dialog box

Edit>Complete Word (keyboard equivalent: Control-B) attempts to compete the current word by searching the buffer for other words beginning with the current word. If there are multiple possible completions, a dialog box is displayed, listing them all. This can be used to enter previously typed long identifiers in program source, for example.

Figure 5-2. The Complete Word dialog box

When editing Lisp source, for example, you might want strings such as "compile-inline" and "after-load-alist" to be treated as one word. To do this, you can set the "noWordSep" buffer-local property, which specifies non-alphanumeric characters which are to be treated as part of words anyway. To set the "noWordSep" buffer-local property, place the following in the first 10 lines of the buffer: 

:noWordSep=-:

Some edit modes have default values for this property; in C mode, for example, the underscore ("_") is treated as a word character. See Chapter 6 for information about edit modes.
