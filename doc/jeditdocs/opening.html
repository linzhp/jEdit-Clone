Chapter 3. Basic File Operations

Opening Files

Before a file can be edited, it must first be opened. When a file is opened, its contents are read into memory and placed in a new buffer. In this manual, a file is the file itself, on disk; a buffer is an open file as it exists in memory, with any unsaved changes, etc.

All open buffers are listed in the Buffers menu; selecting one will display that buffer in the current view. The tabs that run along the top of the text area can also be used to switch between buffers. If you don't like the tabs, they can be disabled in Utilities>Global Options; see the section called The Global Options Dialog Box in Chapter 8 for details.

The title bar of views, as well as the entries in the Buffers menu display various strings depending on the status of the buffer. If "(modified)" is displayed, the buffer has unsaved changes that have not yet been written to disk (or saved). If "(new)" is displayed, the buffer is new; it doesn't exist on disk yet. If "(read only)" is displayed, no changes can be made to that buffer. Markers can be set in read only files but they will not be saved.

Files that you do not have write access to are automatically opened read only. Files can also be forced to open read only from the command line by specifying the -readonly option.

To make a read only buffer read-write, save it. If the save was successful, the read only flag will be cleared (the rationale behind this is that if the save was successful, you have write access to the file, hence you can make changes).

If you edit non-ASCII files, you can select the file encoding used by jEdit in the Utilities>Global Options dialog box; see the section called The Global Options Dialog Box in Chapter 8.

Opening Files From the Command Line

Any file names passed on the command line are opened by jEdit. It is possible to have the caret positioned on a specific line after the buffer is opened. For example, to open paper.tex and go to line number 247, you would invoke: 

$ jedit paper.tex#+247

It is also possible to move the caret to a specific marker. For example, to open Kernel.java and go to the marker named getCommandStack, you would invoke: 

$ jedit Kernel.java#getCommandStack

If your file name includes a dash ("-"), you will need to specify the - - (dash-dash) command line parameter before it so that it is not treated as a command line switch. For example: 

$ jedit - - -file-name-starting-with-a-dash

Opening Files With the Open File Command

File>Open File (keyboard equivalent: Control-O) will display the open file dialog box. Double clicking (or single clicking and then clicking Open) on a file will open it. Double clicking on a directory will list that directory.

The file type popup menu in the open file dialog box contains a list of file types; selecting one will make only files of that type visible in the file list. This is handy when navigating directories with lots of files of different types. The filters can be changed in the File Filters pane of the Utilities>Global Options dialog box; see the section called The Global Options Dialog Box in Chapter 8.

Opening Files With the Open URL Command

File>Open URL (keyboard equivalent: Control-E Control-O) will display the Open URL dialog box. A URL or a file path name (relative path names are prefixed with the current buffer's parent directory) can be entered. Most Java implementations support the ftp: and http: URL protocols, so jEdit can open files from the Internet directly. However, saving to URLs is not supported.

Recently Opened Files

The Recent Files submenu of the File menu lists the 8 most recently opened files. When a buffer is closed, it is added to this list.

Note: The number of recent files can be changed in the Utilities>Global Options dialog box; see the section called The Global Options Dialog Box in Chapter 8.

Current Directory Menu

The Current Directory submenu of the File menu lists all files in the frontmost buffer's directory. Opening files with this menu can be quicker than using the Open File dialog box.

Creating New Files

Sometimes, you want to create a blank, new file, instead of editing an existing one. There are two ways to do this:

  • File>New File (keyboard equivalent: Control-N) will create a new "Untitled" file.

  • Open a non-existent file. If you open a non-existent file using any technique described in this section, a new file with that name will be created when the buffer is first saved.
