Most often, a text editor is used to edit files with some sort of structure, like program source code, or HTML, TeX, and so on. jEdit has several features for making this easier, including edit modes, bracket matching, various indentation commands, auto indent, commands for commenting source, and syntax highlighting.

Edit Modes

Because it is impossible to provide optimal editing for all types of files at once, jEdit provides various edit modes. An edit mode is an editor configuration suitable for editing a specific type of file. Edit modes can specify syntax highlighting, auto indent, and various other settings for editing a file type.

Automatic Mode Selection

When a file is opened, jEdit first checks the file name against a list of known patterns. For example, files whose names end with ".c" are edited in C mode, and files named Makefile are edited in Makefile mode. If a suitable match based on file name cannot be found, jEdit checks the first line of the file. For example, files whose first line is "#!/bin/sh" are edited in shell script mode.

Manually Setting the Edit Mode

If automatic mode selection is not appropriate, the edit mode can be specified manually. To set the current buffer's edit mode on a one-time basis, select the edit mode in the Utilities>Buffer Options dialog box; see the section called The Buffer Options Dialog Box in Chapter 8. To have a buffer open with a specific edit mode every time, set the "mode" buffer-local property to the name of the desired edit mode by placing the following text in one of the first 10 lines of the buffer: 

:mode=edit mode:

Mode nameComments

javascriptJavaScript source code edit mode. Automatically selected for files with extension ".js".
makeMakefile edit mode. Automatically selected for files named "Makefile" or "GNUmakefile".
patchEdit mode for Unix diffs and patches. Automatically selected for files with extension ".diff" or ".patch".
perlPerl edit mode. Automatically selected for files with extension ".pl" or ".pm", or whose first line contains "#!" and "perl".
php3PHP3 edit mode. Automatically selected for files with extension ".php3", ".php", ".phtml" or ".inc".
postscriptPostScript edit mode. Automatically selected for files with extension ".ps".
propsJava property file and MS-DOS INI edit mode. Automatically selected for files with extension ".props" or ".ini", and files named "config.sys" or "properties".
pythonPython script edit mode. Automatically selected for files with extension ".py" or whose first line contains "#!" and "python".
shellUnix shell script edit mode. Automatically selected for files with extension ".sh" or ".csh", and files whose first line is "#!" followed by one of the various Unix shell names.
texTeX edit mode. Automatically selected for files with extension ".tex".
textDefault edit mode. Does nothing special.
tsqlTransact-SQL edit mode. Automatically selected for files with extension ".sql".
xmlXML and SGML source code edit mode. Automatically selected for files with extension "dtd", "rdf", "rss", "sgm", "sgml", "xml", "xsd" or "xsl".
