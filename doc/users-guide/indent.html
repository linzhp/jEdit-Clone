Chapter 4. Editing Source Code

Tabbing and Indentation

A tab is a special character that moves the caret to the next tab stop. Many programming languages rely on tabs to indent lines of source depending on how deeply they are nested in the code. jEdit has several features that make tabs and indentation easier to deal width.

Changing the Tab and Indent Width

jEdit makes a distinction between the tab width, which is is used when displaying tab characters, and the indent width, which is used when a level of indent is to be added or removed, for example by edit mode-specific smart indent routines. Both can be changed in one of several ways:

  • They can be changed on a global or edit mode-specific basis in Editing and Mode-Specific panes of the the Utilities>Global Options dialog box.

  • They can be changed in the current buffer for the duration of the editing session in the Utilities>Buffer Options dialog box.

  • They can be changed in the current buffer for future editing sessions by setting the "tabSize" and "indentSize" buffer-local properties to the desired sizes, by placing the following in one of the first 10 lines of the buffer, where n is the desired tab width, and m is the desired indent width: 

    :tabSize=n:indentSize=m:

Soft Tabs

Because files indented using tab characters may look less than ideal when viewed on a system with a different default tab size, it is sometimes desired to use multiple spaces, known as soft tabs, instead of real tabs to indent code.

There are three ways to enable or disable soft tabs:

  • Soft tabs can be enabled or disabled on a global or edit mode-specific basis in the Editing and Mode-Specific panes of the Utilities>Global Options dialog box.

  • Soft tabs can be enabled or disabled in the current buffer for the duration of the editing session in the Utilities>Buffer Options dialog box.

  • Soft tabs can be enabled or disabled in the current buffer for future editing sessions by placing the following in one of the first 10 lines of the buffer, where flag is either "true" or "false": 

    :noTabs=flag:

Changing the soft tabs setting has no effect on existing tab characters; it merely affects newly-inserted tabs. To convert existing hard tabs to soft tabs and vice versa, see the commands in the next section.

Indentation-Related Commands

Edit>Lines>Shift Indent Left (keyboard equivalent: Alt-Left) and Edit>Lines>Shift Indent Right (keyboard equivalent: Alt-Right) will add or remove, respectively, one level of indent to every line in the selection, or the current line if nothing is selected.

Edit>Selection>Spaces to Tabs and Edit>Selection>Tabs to Spaces convert soft tabs to hard tabs, and vice versa, respectively.

Automatic Indent

Pressing Tab or Space at the start of every line to create the appropriate indent is very tedious, so jEdit has an auto indent feature.

Edit>Lines>Indent Selected Lines (keyboard equivalent: Control-I) indents all selected lines, or the current line if there is no selection. If indent on enter is enabled, pressing Enter will create a new line with the appropriate amount of indent automatically. If indent on tab is enabled, pressing Tab on an unindented line will insert the appropriate amount of indentation. Pressing it again will insert a tab character.

By default, indent on enter is enabled and indent on tab is disabled. There are three ways to specify when indentation should be performed:

  • Indent on enter and indent on tab can be set on a global or edit mode-specific basis in the Editing and Mode-Specific panes of the Utilities>Global Options dialog box.

  • Indent on enter and indent on tab can be set in the current buffer for the duration of the editing session in the Utilities>Buffer Options dialog box.

  • Indent on enter and indent on tab can be set in the current buffer for future editing sessions by placing the following in the first 10 lines of a buffer, where flag is either "true" or "false": 

    :indentOnEnter=flag:indentOnTab=flag:

Auto indent rules are mode-specific. In most edit modes, the indent of the previous line is simply copied over. However, in C-like languages (C, C++, Java, JavaScript), if the previous line contains an open curly bracket ("{"), an extra tab (or soft tab) is inserted. Similarly, when you type a closing curly bracket ("}"), a level of indentation is removed. Also, "if", "else", "while" and so on create an extra level of indent on the next line.

If you want to insert a literal tab or newline without performing indentation, prefix the tab or newline with Control-E V. For example, to create a new line without any indentation, type Control-E V Enter.
