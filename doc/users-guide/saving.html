Chapter 2. Working With Files

Saving Files

Changed made to a buffer leave the disk file unchanged until the buffer is saved. File>Save (shortcut: Control-S) saves the current buffer to disk. File>Save All Buffers (shortcut: Control-E Control-S) will save all buffers to disk. File>Save As and File>Save a Copy As prompt for an alternate file name to save the buffer to. The two commands differ in that Save As renames the buffer, so that subsequent saves will use the new path, whereas Save a Copy As doesn't, so subsequent saves will continue to use the old path. Also, the Save a Copy As command doesn't clear the buffer's "modified" flag.

Autosave and Backups

jEdit's autosave feature protects your work from computer crashes and such. Every 30 seconds, all buffers with unsaved changes are written out to their respective file names, enclosed in hash ("#") characters. For example, program.c will be autosaved to #program.c#. Because saving a buffer using one of the commands in the previous section automatically deletes the autosave file, they will never be visible during normal operation. However, if jEdit crashes before you have a chance to save, the autosave file will remain, along with your unsaved changes, and jEdit will offer to recover the autosaved data next time the original file is opened.

Another safety feature is file backups. When a buffer is saved for the first time after being opened, its original contents are preserved in buffer's file name suffixed with a tilde ("~"). For example, paper.tex will get backed up to paper.tex~. Backups can be used to roll back to the previous version of a file after changes were made.

The autosave and backup features can be configured in the Loading and Saving pane of the Utilities>Global Options dialog box.

Line Separator Characters

The three major operating systems use different conventions to mark line endings in text files. The MacOS uses Carriage-Return characters (\r, ^M) for that purpose. Unix uses Newline characters (\n, ^J). DOS and Windows use both (\r\n, ^M^J). jEdit can read and write files in all three formats. The line separator of the current buffer can be changed in the Utilities>Buffer Options dialog box. The default for new files can be changed in the Editing pane of the Utilities>Global Options dialog box.

When Another Application Modifies the File

If an open buffer is modified on disk by another application, a warning dialog box is displayed, offering to either continue editing (and lose changes made by the other application) or reload the buffer from disk. This feature can be disabled in the General pane of the Utilities>Global Options dialog box.

File>Reload can be used to reload the current buffer from disk at any other time; a confirmation dialog box will be displayed first if the buffer has unsaved changes. File>Reload All Buffers discards changes in all open buffers and reload them from disk, asking for confirmation first..
