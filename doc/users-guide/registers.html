Chapter 3. Editing Text

Registers

Each register can hold either a text string or caret position for later use. Registers have single-character names, hhence their number is limited by how many keys can be typed on your keyboard. Register contents are global to the editor; all buffers and views share the same set. Registers are not persistent and their contents are lost when jEdit exits. The register $ is an alias for the clipboard, and therefore registers can be considered as an extension of the clipboard concept.

Edit>Registers>View Registers displays a dialog box for viewing register contents. The popup menu lists all defined registers; selecting one will display its contents and type (text or position). It is not possible to change or add registers in this dialog box; it is only for viewing.

All register commands except for View Registers listen for the next key press and operate on the register with that name. For example, to copy the selection to register X, you would press Control-R Control-C X (The character "x" will not be inserted into the buffer).

Text Registers

Edit>Registers>Cut to Register (shortcut: Control-R Control-X) reads the next character entered at the keyboard, and stores the selected text in that register, removing it from the buffer.

Edit>Registers>Copy to Register (shortcut: Control-R Control-C) reads the next character entered at the keyboard, and stores the selected text in that register, leaving it in the buffer.

Edit>Registers>Append to Register (shortcut: Control-R Control-A) reads the next character entered at the keyboard, and appends the selected text to that register, leaving the original text in the buffer.

Edit>Registers>Paste from Register (shortcut: Control-R Control-V) reads the next character entered at the keyboard, and replaces the selection with the contents of that register.

Position Registers

Edit>Registers>Save Position to Register (shortcut: Control-T) reads the next character entered at the keyboard, and stores the current buffer name and caret position in that register. Because buffer names are saved in position registers, they can be used to quickly flip between multiple buffers.

Edit>Registers>Go to Position in Register (shortcut: Control-Y) reads the next character entered at the keyboard, opens the buffer named in that register (if necessary), and moves the caret to the saved position.

Edit>Registers>Select to Position in Register (shortcut: Control-U) reads the next character entered at the keyboard, and creates a selection from the current caret position to the offset specified in that register.

Edit>Registers>Swap Position with Register (shortcut: Control-K) reads the next character entered at the keyboard, and swaps the caret position and current buffer with that entered in the register.

Note: Caret positions cannot be saved to the $ register (clipboard).
