Chapter 8. Virtual File Systems

The FTP Filesystem

jEdit has its own FTP protocol implementation which is superior to the one that ships with Java. Unlike the default implementation, it supports non-anonymous logins and lets you browse remote filesystems.

To open a file stored on an FTP server, select the Connect to FTP Server command from the More menu in the VFS browser. You will be prompted for a host name, user name and password.

Figure 8-1. The FTP login dialog box

If a buffer was loaded from an FTP server, File>Save (keyboard equivalent: Control-S) will automatically upload it. To upload a buffer not originally from an FTP server, invoke File>Save As and perform the above steps to connect to the FTP server.

Some jEdit features do not work when editing remote files; no autosave or backup files are created, and the File>Current Directory menu cannot currently list remote directories.

To improve performance, jEdit caches directory listings from remote FTP servers. Adding and deleting files from within jEdit will automatically update the cache when necessary, but if you use another FTP client, file listings may become out of date. To flush the cache, invoke Utilities>Flush Remote Directory Cache. jEdit also remembers passwords on remote servers until it is closed. To force jEdit to forget saved passwords without quitting and restarting it, invoke Utilities>Forget Remote Passwords.

Tip: If you know the exact path name of a buffer you want to open via FTP, you can forgo the VFS browser by entering an FTP URL (ftp://user@server/path) directly in the File>Open Path dialog box. You will then be prompted for a password, and the file will be loaded.
