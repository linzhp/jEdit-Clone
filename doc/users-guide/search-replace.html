Chapter 3. Editing Text

Search and Replace

The Search and Replace Dialog Box

Search>Find (keyboard equivalent: Control-F) displays the search and replace dialog box. Search>Find Selection (keyboard equivalent: Control-E Control-F) displays the search and replace dialog box and with the currently selected text in the Search for field.

The Search for: and Replace with: field store previously entered strings; see Appendix B.

Unless the Keep Dialog check box is selected, the dialog box will automatically close after a successful search.

If the Ignore Case check box is selected, the search will be case insensitive; for example, entering "Hello" in the search field will locate "hello", "HELLO", and "HeLlO"

If the Regexp check box is selected, the search string will be interpreted as a regular expression. Regular expressions can be used to match inexact sequences of text; see Appendix E for details.

Clicking the Multi File button displays a dialog box chosing the file set to search. Keep in mind that searches over a large number of files may consume a large amount of memory, and/or take a long time to finish.

Clicking Find or pressing Enter selects the next occurence of the search string.

Clicking Replace replaces all occurrences of the search string in the selection with the replacement string. Clicking Replace & Find will do the above, and then and locate the next occurrence of the search string. Clicking Replace All replaces all occurrences of the search string with the replacement string.

Tip: When regular expressions are enabled, the string "\n" will match newlines. This can be used to perform multiple-line search and replace.

The HyperSearch Dialog Box

The HyperSearch dialog box lists all lines in a buffer (or a set of buffers, if multiple file search is enabled) that contain a specified string.

Search>HyperSearch (keyboard equivalent: Control-H) displays the HyperSearch dialog box. Search>HyperSearch For Selection (keyboard equivalent: Control-E Control-H) displays the HyperSearch dialog box and searches for all occurrences of the currently selected text.

The Ignore Case, Regexp and Multi File buttons behave exactly as they do in the search and replace dialog box.

Clicking the Start button, or pressing Enter will begin the search. Clicking Stop aborts a search in progress.

Once a search has been performed, the result are displayed in a tree, with matches from different buffers shown in dufferent branches of the tree. Clicking on a match will open the respective buffer and move the caret to the line containing the search string.

The Search Bar

The "search bar" provides quick access to searching functions. To search for a string, enter it into the search bar and press Enter. The search bar stores previously entered strings; see Appendix B for details. If Enter is pressed with no string entered, the Search and Replace dialog box will be displayed. Pressing Escape will return the keyboard focus back to the text area.

If the Incremental check box is selected, the search string will be searched for immediately as it is being typed; otherwise, the search will only commence when Enter is pressed. Note that when incremental search is active, the Multi File button is grayed out and multiple file searching becomes unavailable.

Search>Quick Search (keyboard equivalent: Control-,) activates the search bar. Search>Incremental Search (keyboard equivalent: Control-.) is like the Quick Search activates the search bar and selects the Incremental check box.

The search bar can be disabled in the General pane of the Utilities>Global Options dialog box.
